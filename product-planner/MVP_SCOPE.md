# MVP 范围定义：PiP 弹幕浏览器插件

## 1. 范围拆解（Must / Should / Could）

### Must（首发必须有）
- 平台与站点
  - Chromium 浏览器（Chrome/Edge）支持。
  - Bilibili 点播页面支持（普通视频）。
  - YouTube 点播页面基础支持（基础弹幕/评论流映射，能力以稳定性优先）。
- 核心功能
  - 一键打开 PiP 弹幕模式。
  - PiP 窗口内弹幕叠加显示（滚动/顶部/底部基础模式）。
  - 与视频状态同步：播放、暂停、seek、倍速。
  - 基础设置：字号、透明度、弹幕速度、显示区域。
  - 基础过滤：关键词屏蔽、重复弹幕合并（短时间窗口）。
- 稳定与诊断
  - 页面刷新/换集后可自动恢复挂载。
  - 基础日志开关（仅本地）用于问题排查。
  - 默认开启轻量模式（限帧 + 弹幕密度上限）。

### Should（首发可有）
- PiP 内快捷操作（开关弹幕、清屏）。
- 样式预设（简洁、标准）。
- 弹幕遮挡优化（尽量避开字幕区域）。
- YouTube 场景兼容增强（页面切换和恢复稳定性）。

### Could（后续增强）
- Bilibili 直播弹幕支持（实时流）。
- YouTube 直播聊天映射为弹幕。
- 高级过滤规则（正则、消息类型）。

## 2. 明确不做项（MVP 不包含）
- 不做 Firefox / Safari 首发支持。
- 不做云端用户数据存储与账号系统。
- 不做付费系统、订阅系统、运营后台。
- 不做视频录制、弹幕导出（ASS/视频硬字幕）能力。
- 不做 AI 去噪、智能推荐等高计算开销能力。

## 3. 需求验收标准（MVP）
- 体验验收
  - 用户在 Bilibili 点播页点击插件按钮后，3 秒内进入 PiP 弹幕模式。
  - seek 后 1 秒内完成弹幕重同步，视觉偏差在可接受范围。
- 稳定验收
  - 连续 60 分钟播放无白屏、无卡死、无明显弹幕堆积。
- 性能验收
  - CPU/内存占用达标（以 PRODUCT_BRIEF 指标为准）。
  - 低端设备模式下保持可用交互，不出现明显卡顿。
- 兼容验收
  - 至少覆盖 Bilibili 常见页面模版（普通稿件页、分P切换）。
  - 覆盖 YouTube 常见点播页面模版（单视频页、同页切换）。

## 4. 首周可落地动作（Week 1）
- 搭建 MV3 插件骨架（background/content/popup/options）。
- 实现 Bilibili 视频元素与播放状态探测。
- 实现 YouTube 视频元素与播放状态探测（仅最小可用路径）。
- 接入 Document PiP，验证自定义弹幕层可渲染。
- 定义统一弹幕事件模型（DanmakuEvent）并打通最小链路：
  - 采集 -> 标准化 -> 调度 -> 渲染。
- 完成 5 个核心手测用例脚本：
  - 进入 PiP、暂停、seek、切分P/换视频、关闭重开。
